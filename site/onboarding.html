Telegram onboarding

Set up a channel, bot, and TrackCast in a few minutes.

This guide is written for testers who have never configured a Telegram bot before. Follow it once, then TrackCast can reuse the same channel for future sets.

Before you start

Decide whether your broadcast will be public or private. Public channels can use an @channelname. Private channels usually need a numeric -100... chat ID.

Step 01

Create or choose a Telegram channel.

TrackCast posts into a channel, so create one for your live set updates or reuse an existing test channel.

  • Open Telegram and create a new channel.
  • Give it a clear name, like TrackCast Test.
  • Make it public if you want a simple @username.

Step 02

Create the bot with BotFather.

BotFather gives you the API token TrackCast needs to send messages on your behalf.

  • Search Telegram for @BotFather.
  • Send /newbot.
  • Choose a name and a username ending in bot.

Step 03

Copy the API token.

The token is the password for the bot. Keep it private and paste it only into TrackCast.

  • BotFather returns a long token after the bot is created.
  • Copy the whole token, including the colon in the middle.
  • If it leaks later, regenerate it in BotFather.

Step 04

Add the bot as a channel admin.

The bot cannot post to your channel until Telegram gives it permission.

  • Open your channel settings.
  • Add the bot as an administrator.
  • Enable permission to post messages.

Step 05

Paste the token and channel into TrackCast.

Open TrackCast, go to Setup, paste the bot token, then enter the channel destination.

  • Public channel: use @channelusername.
  • Private channel: use the numeric -100... chat ID.
  • Click the test action before going live.

Telegram verified
Step 06

Start a short test broadcast.

Play a track in your DJ software, connect TrackCast, then start broadcasting. Your channel should receive the live messages.

  • Use a private test channel for the first run.
  • Confirm the first track is correct before inviting people.
  • Stop the broadcast when the test is done.
